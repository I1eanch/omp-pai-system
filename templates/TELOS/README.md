# TELOS Starter

TELOS хранит устойчивые цели, проекты, убеждения, решения, идеи, вызовы и извлечённые уроки пользователя. Starter-файлы намеренно не содержат примеров с людьми, организациями, датами, контактами или личными обстоятельствами.

## Lifecycle

1. `init` копирует отсутствующие starter-файлы в `${dataRoot}/TELOS`.
2. Пользователь заполняет `Entries` локально.
3. Изменение `status` на `active` означает, что файл содержит рабочие записи.
4. Удаление и upgrade не перезаписывают пользовательские значения.
5. Private export включает TELOS только после явного локального вызова.

Frontmatter каждого файла проверяется по `schema.json`. Поле `updated` остаётся `null` до первого пользовательского изменения. Записи оформляются как Markdown-секции или bullets по выбору пользователя; package не навязывает персональные значения.
